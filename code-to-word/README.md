# code-to-word

将代码文件批量转换为带语法高亮的 Word 文档，适合课程作业提交、论文附录整理、代码归档和给非开发人员查看喵～

## 功能特性

- 扫描脚本所在目录下的代码文件（自动排除脚本自身，不会递归子目录）
- 自动识别文件语言并应用语法高亮
- 输出为 `.docx` Word 文档
- 统一使用 `Consolas` 等宽字体
- 默认字号约为 `10.5pt`
- 批量处理多个文件，逐个输出成功或失败结果

## 适用场景

- 需要把源码整理进课程实验报告
- 需要将算法实现导出到 Word 便于打印或提交
- 需要给不方便直接看源码的人员提供可读版本
- 需要快速归档当前目录下的一批代码文件

## 依赖要求

- Python 3.10+
- `pygments`
- `spire.doc`

安装命令：

```bash
pip install pygments spire.doc
```

## 当前支持的代码文件类型

脚本当前会处理以下扩展名：

- `.m`
- `.py`
- `.c`
- `.cpp`
- `.java`
- `.cs`
- `.js`
- `.ts`
- `.go`
- `.rs`

如果文件扩展名无法被 `Pygments` 正确识别，脚本会回退到 `MatlabLexer` 进行高亮处理。


## 使用方法

### 1. 准备代码文件

将需要转换的代码文件和 `gen_word.py` 放在同一个目录下。

例如：

```text
code-to-word/
├── gen_word.py
├── main.py
├── utils.cpp
└── test.m
```

### 2. 运行脚本

在 `code-to-word` 目录中执行：

```bash
python gen_word.py
```

### 3. 查看输出结果

脚本会在当前目录下生成同名的 `.docx` 文件。

例如：

```text
main.py    -> main.docx
utils.cpp  -> utils.docx
test.m     -> test.docx
```

## 运行输出示例

```text
找到 3 个代码文件，开始转换...

[OK] main.py -> main.docx
[OK] test.m -> test.docx
[OK] utils.cpp -> utils.docx

完成！文件保存在: D:\GitHub\SrP-Scripts\code-to-word
```

如果某个文件处理失败，则会输出类似：

```text
[FAIL] main.py: 'utf-8' codec can't decode byte ...
```

## 工作机制说明

脚本执行流程如下：

1. 获取脚本所在目录
2. 扫描目录下所有受支持的代码文件（自动排除脚本自身）
3. 读取文件内容（当前固定使用 UTF-8 编码）
4. 根据文件名自动选择语法高亮规则
5. 使用 `Pygments` 生成带高亮的 RTF 内容
6. 使用 `Spire.Doc` 写入 `.docx` 文档

这意味着它当前是一个“同目录批处理工具”，不会递归扫描子目录，也不会处理目录外的文件。

## 输出规则

- 输出目录：脚本所在目录
- 输出文件名：源码文件名去掉扩展名后加 `.docx`
- 文档格式：`Docx2016`
- 代码字体：`Consolas`
- 字号：约 `10.5pt`

## 注意事项

- 脚本只扫描当前目录，不会递归处理子文件夹
- 脚本会自动排除自身（`gen_word.py`），不会将自身转换为 `.docx`
- 读取源码时固定使用 `UTF-8`，如果文件是 `GBK`、`ANSI` 等编码，可能报错
- 当前没有命令行参数，处理范围和输出路径都写死在脚本逻辑中
- 输出文件名只保留主文件名，不包含原扩展名
- 如果目录里没有支持的代码文件，脚本会直接提示“未找到代码文件”

## 常见问题

### 1. 运行时报 `ModuleNotFoundError`

说明依赖没有安装完整，执行：

```bash
pip install pygments spire.doc
```

### 2. 提示编码错误

**现象：**

```text
[FAIL] xxx.py: 'utf-8' codec can't decode ...
```

**原因：**

源码文件不是 UTF-8 编码。

**解决方式：**

- 先用编辑器将源码另存为 UTF-8
- 或后续自行修改脚本，增加多编码兼容处理

### 3. 某些语言高亮效果不准确

**原因：**

脚本是按文件名扩展名识别语言的；识别失败时会回退到 Matlab 词法器。

**解决方式：**

- 确认文件扩展名正确
- 如有需要，可在脚本中手动指定 lexer

### 4. 为什么没有处理子目录文件

因为当前脚本使用的是当前目录文件扫描逻辑，只处理 `code-to-word` 目录下的文件，不递归子目录。这是现有实现的设计行为，不是异常。
